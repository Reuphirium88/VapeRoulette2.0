Контекст проекта:

Нужно сделать Telegram Mini App (WebApp) с системой лояльности и лутбоксами для магазина. Фронтенд — статический HTML/CSS/JS на GitHub Pages. Бэкенд — Python 3.11 + FastAPI + SQLModel + SQLite, локально, с возможностью пробросить наружу через туннель (ngrok / Cloudflare Tunnel). В будущем бэкенд переедет на VPS/домен без смены API. Модульная архитектура: отдельные домены users, loyalty (XP/transactions), lootboxes, отдельные роутеры api/client и api/admin, отдельные модули db, security, config.

Требования:
- Всё должно быть максимально модулярно и соответствовать best practice
- НЕ использовать Docker, НЕ использовать Prisma.
- Код писать аккуратно, разбивать на файлы: app/main.py, app/api/client.py, app/api/admin_*.py, app/domain/users/*.py, app/domain/loyalty/*.py, app/domain/lootboxes/*.py, app/db.py, app/schemas.py, app/security.py и т.п.
- В роутерах никакого сырого SQL, только вызовы доменных сервисов и работу через SQLModel/Session.
- Фронт: чистый index.html + app.js + styles.css, без сборщиков.
- Telegram WebApp SDK подключён в index.html, но приложение должно уметь работать и в обычном браузере с моковыми данными, если Telegram.WebApp недоступен.

- сначала обнови/создай файлы,
- затем кратко опиши, что добавлено и как запускать/проверять.
